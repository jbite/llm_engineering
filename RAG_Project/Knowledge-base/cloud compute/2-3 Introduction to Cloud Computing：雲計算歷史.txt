# 雲計算的歷史和演進
雲計算的歷史和演進是一個持續進化的技術旅程。在這篇博文中，我們將逐步分析雲計算的歷史和發展過程，從早期的大型機到現代的雲計算平台。

## 雲計算的起源
### **1950年代：大型機時代**
- **大型機**：雲計算的概念可以追溯到1950年代，當時擁有高容量處理能力的大型機變得普及。
- **時間共享/資源池**：時間共享（或稱資源池）的實踐演化出來，以高效利用大型機的計算能力。
- **使用方式**：通過僅用於訪問大型機的簡單終端（dumb terminals），多個用戶可以從任何終端訪問相同的數據存儲層和 CPU 功能。
### **1970年代：虛擬機（VM）的推出**
- **虛擬機操作系統**：1970年代，隨著稱為虛擬機（VM）的操作系統的推出，使得在一個物理節點上可以有多個虛擬系統或虛擬機。
- **技術演變**：這種虛擬機操作系統從1950年代共享主機的應用演變而來，允許在同一硬件上存在多個獨立的計算環境。
- **虛擬化的驅動**：每個虛擬機托管客戶操作系統，儘管資源被共享，但表現得像擁有自己的內存、CPU和硬盤一樣。虛擬化成為了通信和計算中一些最重要演變的關鍵技術驅動因素。
## **20年前：物理硬件的價格**
- **昂貴的物理硬件**：即使是在20年前，物理硬件也是相當昂貴的。
- **互聯網和虛擬化**：隨著互聯網變得更加可訪問，以及為了使硬件成本更具經濟實用性，伺服器被虛擬化到共享托管環境、虛擬私人伺服器和虛擬專用伺服器。
## **現代雲計算的崛起**
- **虛擬化的進步**：隨著技術和虛擬化監控器的改進，能夠可靠地共享和交付資源，一些公司決定使雲的好處對用戶可訪問。
- **即時創建實例**：由於伺服器已經在線，創建一個新實例是瞬間的。
- **按需支付**：用戶現在可以從更大的可用資源池中訂購雲資源，並按使用情況付費。
- **實用計算模型**：這種按需支付或實用計算模型成為推動雲計算發展的關鍵因素之一。
- **運營支出（OpEx）模型**：這種模式允許公司和開發者按使用情況支付計算資源，就像電力單位一樣，從而轉向更現金流友好的運營支出模型。
- **規模調整**：這也使他們能夠在使用高峰期擴展工作負載，並在使用減少時縮減規模。

現代雲計算的誕生，使得各種規模的公司，無論是擁有少量或沒有硬件，還是擁有大量硬件的公司，都因為現在不再需要大量資本開支來購買硬件，而可以根據需要支付計算資源費用而受益匪淺。
